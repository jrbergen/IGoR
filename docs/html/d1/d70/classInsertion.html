Evaluate all Event_Realization of a RecEvent for a given sequence.

The iterate method is the heart of IGoR's scenario exploration. Model_Parms define an order in which the RecEvent should be processed. Upon call of iterate all EventRealization of the RecEvent are assessed and for each possible realization the iterate method is called recursively for the next event. Inside the iterate method a filtering on too improbable realizations is performed (tree prunning) using the downstream_proba_map.

The index map is used to read off the probability of the current RecEvent EventRealization at the correct location given the events parent's realizations. It is further modified to take into account the current event's realization when its children realization probabilities will be read.
